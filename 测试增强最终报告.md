### 为什么总体覆盖率是7%？

**计算公式**:
```
总体覆盖率 = 已测试类的覆盖率 × (已测试类数 / 总类数)
          = 71% × (5 / 102)
          = 71% × 5%
          = 7%
```

**这是正常的**，因为：
1. 项目有102个类
2. 只测试了5个类（5%）
3. 97个类未测试

**解决方案**: 增加测试覆盖更多类

## 📋 使用指南

### 运行测试

```bash
# 运行所有测试
cd backend
mvn test

# 运行单个测试类
mvn test -Dtest=JwtUtilTest

# 运行单个测试方法
mvn test -Dtest=JwtUtilTest#testGenerateAccessToken

# 生成覆盖率报告
mvn clean test
# 报告位置: target/site/jacoco/index.html
```

### 查看覆盖率报告

```bash

open backend/target/site/jacoco/index.html

# Linux
xdg-open backend/target/site/jacoco/index.html
```

2. 查看CSV报告：

## 🎓 经验总结

### 成功经验

1. **渐进式测试**
   - 先测试核心功能
   - 逐步扩大覆盖范围
   - 保证测试质量

2. **Mock隔离**
   - 不依赖外部资源
   - 测试执行快速
   - 结果可重复

3. **文档先行**
   - 测试报告详细
   - 配置说明清晰
   - 便于后续维护

### 遇到的挑战

1. **实体类字段名不匹配**
   - 解决：使用grep_code查看实际定义
   - 教训：以源码为准，不要假设

2. **Spring Boot版本兼容**
   - 解决：使用正确的MockBean导入
   - 教训：注意版本差异

3. **覆盖率理解**
   - 解决：区分总体覆盖率和模块覆盖率
   - 教训：设定合理目标
