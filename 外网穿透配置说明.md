# 🚀 外网穿透配置说明

## 当前配置

- **后端API**：通过NATAPP穿透到外网
  - 外网地址：http://vc9abe67.natappfree.cc
  - 本地端口：8082
  
- **前端**：本地运行，不穿透
  - 本地端口：3000
  - API调用：/api (相对路径，由NATAPP代理)

## 工作原理

```
外网用户
  ↓
http://vc9abe67.natappfree.cc (NATAPP穿透后端8082)
  ↓
/api/auth/admin-login (前端请求相对路径)
  ↓
http://vc9abe67.natappfree.cc/api/auth/admin-login (NATAPP转发)
  ↓
localhost:8082/api/auth/admin-login (后端Spring Boot)
```

## 启动步骤

### 1. 启动后端（如果未运行）
```powershell
cd d:\Myproject\Course\backend
mvn spring-boot:run
```

### 2. 启动NATAPP穿透（前端3000端口）
```powershell
cd d:\Myproject\Course
.\natapp\natapp.exe -authtoken=2789714503e149f2 -subdomain=vc9abe67 -log=stdout
```

### 3. 构建并启动前端（使用NATAPP域名）
```powershell
cd d:\Myproject\Course\frontend
# 修改.env.production中的VITE_API_BASE_URL为NATAPP域名
# VITE_API_BASE_URL=http://vc9abe67.natappfree.cc/api
npx vite build
serve -s dist -l 3000
```

### 或使用一键脚本
双击运行：`重启所有服务.bat`

## 注意事项

1. 后端必须在本地运行（8082端口）
2. NATAPP保持运行
3. 前端服务保持运行
4. 所有服务在同一台机器上

## 为什么这样配置？

- ✅ NATAPP免费版只支持1个隧道
- ✅ 前端和后端在同一台机器
- ✅ 前端通过localhost直接调用后端
- ✅ 无需额外配置跨域（后端已允许localhost）
