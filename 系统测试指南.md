## 🚨 重要提示

后端启动时发现端口8082被占用，需要先停止占用进程。

---

## 📋 测试前准备

### 1. 停止占用端口的进程

**Windows系统**:
```powershell
# 查找占用8082端口的进程
netstat -ano | findstr :8082

# 停止进程（替换PID为实际进程ID）
taskkill /F /PID <进程ID>
```

**或者修改后端端口**:
编辑 `backend/src/main/resources/application-dev.yml`，修改端口为8083：
```yaml
server:
  port: 8083
```

### 2. 确保MySQL服务运行

```powershell
# 检查MySQL服务状态
sc query MySQL80

# 如果未启动，启动MySQL
net start MySQL80
```

## 🚀 启动步骤

### 方式一：使用启动脚本（推荐）

双击运行项目根目录的 `start.bat` 文件。

### 方式二：手动启动

#### 1. 启动后端服务

```powershell
cd d:\Myproject\Course\backend
mvn spring-boot:run
```

**等待看到以下信息表示启动成功**:
```

```

#### 2. 启动前端服务

打开新的终端窗口：
```powershell
cd d:\Myproject\Course\frontend
npm run dev
```

**等待看到以下信息表示启动成功**:

## 🐛 常见问题排查

### 问题1: 后端启动失败 - 端口被占用
**症状**: 
```
Web server failed to start. Port 8082 was already in use.
```

**解决方案**:
1. 停止占用端口的进程（见上文）
2. 或修改application.yml中的端口号

### 问题2: 前端启动失败 - 依赖未安装
**症状**:
```
Error: Cannot find module 'xxx'
```

**解决方案**:
```powershell
cd frontend
npm install
```

### 问题3: API请求失败 - 401 Unauthorized
**症状**: 前端请求返回401错误

**解决方案**:
1. 检查是否已登录
2. 检查token是否过期
3. 重新登录获取新token

### 问题4: 数据库连接失败
**症状**:
```
Could not open JDBC Connection for transaction
```

**解决方案**:
1. 检查MySQL服务是否启动
2. 检查application-dev.yml中的数据库配置
3. 确保数据库已创建

### 问题5: 页面空白
**症状**: 前端页面显示空白

**解决方案**:
1. 打开浏览器开发者工具（F12）
2. 查看Console标签是否有错误
3. 查看Network标签检查API请求
4. 确认后端服务是否正常启动
