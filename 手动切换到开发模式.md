# 🚀 手动切换到开发模式

## 当前状态

- ✅ 后端：运行中（8082端口）
- ✅ 前端开发模式：运行中（3001端口）
- ⚠️ NATAPP：需要重启（穿透3001端口）

---

## 操作步骤

### 第1步：关闭旧的NATAPP窗口

找到标题为 **"NATAPP穿透"** 的窗口，**直接关闭**它。

或者在任务管理器中：
1. 按 `Ctrl + Shift + Esc` 打开任务管理器
2. 找到 `natapp.exe` 进程
3. 右键 → 结束任务

---

### 第2步：启动新的NATAPP（穿透3001端口）

打开**新的PowerShell窗口**，运行：

```powershell
cd d:\Myproject\Course
.\natapp\natapp.exe -authtoken=2789714503e149f2 -log=stdout
```

**注意**：这次不加 `-config` 参数，直接穿透3001端口。

---

### 第3步：查看外网地址

NATAPP启动后会显示类似：
```
Tunnel Status         Online
Forwarding            http://xxxxx.natappfree.cc -> http://127.0.0.1:3001
```

**记录下这个新的域名**（可能不是vc9abe67了）

---

### 第4步：更新前端API配置

由于域名可能变化，需要修改前端配置：

1. 打开文件：`frontend\.env.production`
2. 修改API地址为新的NATAPP域名：
   ```
   VITE_API_BASE_URL=http://新域名.natappfree.cc/api
   ```
3. 保存文件

**或者更简单**：暂时使用局域网访问

---

### 第5步：访问

**局域网访问**（推荐，无需配置）：
```
http://192.168.1.147:3001/admin/login
```

**外网访问**（需要完成第2-4步）：
```
http://新域名.natappfree.cc/admin/login
```
